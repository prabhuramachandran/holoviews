InstallationÂ¶

HoloViews is compatible with Python 2.7, 3.3, and 3.4.

HoloViews requires Param and Numpy, neither of which has any required dependencies, and so it should be very easy to integrate HoloViews into your workflow or as part of another project.

For plotting, HoloViews uses Matplotlib, which most scientists and engineers using Python will already have installed. HoloViews is pure Python, but it also provides optional extensions enabled with %load_ext above that make it integrate well with IPython Notebook 2 and 3.

Matplotlib and IPython Notebook can be installed using your operating system’s package manager, as part of a scientific Python distribution like Anaconda (particularly convenient on systems shipped without pip, such as Windows or Mac), or by using pip:

pip install matplotlib 'ipython[notebook]'

You can then obtain the latest public release of HoloViews and its core dependencies (Param and Numpy) using pip:

pip install holoviews

We also support the following install option:

pip install 'holoviews[extras]'

In addition to HoloViews and its required dependencies, [extras] installs the optional mpld3, pandas and Seaborn libraries.

To get the latest development version you can instead clone our git repositories:

git clone git://github.com/ioam/param.git
git clone git://github.com/ioam/holoviews.git

Once you’ve installed HoloViews, you can get started by launching IPython Notebook:

ipython notebook

Now you can download the tutorial notebooks, unzip them somewhere IPython Notebook can find them, and then open the Homepage.ipynb tutorial or any of the others in the Notebook. Enjoy exploring your data!

ContributorsÂ¶

HoloViews was developed by Jean-Luc R. Stevens and Philipp Rudiger, in collaboration with James A. Bednar.

HoloViews is completely open source, available under a BSD license freely for both commercial and non-commercial use. HoloViews is designed to be easily extensible, and contributions from users are welcome and encouraged. In particular, HoloViews components can be combined in an infinite number of ways, and although the tutorials are tested continuously, it is impossible for us to test all other conceivable combinations. Thus we welcome bug reports and feature requests, particularly if they come with test cases showing how to reproduce the bugs and pull requests showing how to fix the bug or implement the feature!
